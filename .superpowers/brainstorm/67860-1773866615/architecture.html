Architecture: Slack as Terminal

How the pieces fit together

Slack Workspace
│
├─ @SlackClaw run: fix the login crash ← trigger in #dayist-dev
│
▼
bot_unified.py (SlackClaw Bot)
│
├─ Detects "run:" prefix → creates SlackSession
├─ Spawns backend based on configured mode
│
│ ┌─────────────────────────────────────┐
│ │ SessionBackend │
│ │ │
│ │ MaxBackend APIBackend │
│ │ claude -p anthropic.stream │
│ │ --continue messages.create │
│ └─────────────────────────────────────┘
│
├─ Streams output → chunks → posts to Slack thread
│
├─ When input needed:
│ ├─ Posts Block Kit buttons (structured choices)
│ └─ Also listens for typed thread replies
│
├─ User responds (button click OR typed reply)
│ └─ Feeds back to session → continues
│
└─ Session ends → posts ✅ summary

Config (persisted in .env)
├─ SESSION_BACKEND=max or SESSION_BACKEND=api
├─ Changed via: @SlackClaw set mode max
└─ Set during onboarding on first bot start
All SlackClaw AI calls use the same backend — quick @mentions and run: sessions both go through MaxBackend or APIBackend. No split billing.
